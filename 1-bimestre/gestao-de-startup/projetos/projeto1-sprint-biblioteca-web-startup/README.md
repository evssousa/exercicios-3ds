# 🚀 PROJETO 1 - SPRINT BIBLIOTECA: WEB STARTUP

# 👥 Divisão de Papéis

Cada grupo (4–5 alunos):

- 👑 **Product Owner** → Define problema e funcionalidades
- 💻 **Dev Web** → Trabalha no computador (rodízio)
- 🎨 **UX Designer** → Desenha telas no papel
- 📊 **Analista de Sistemas** → Escreve requisitos e fluxo
- 🧠 **Gestor** (opcional se tiver 5) → Organiza tempo e equipe

# ⏱️ ROTEIRO

## 🔹 Pensamento

> “Com apenas 1 computador por equipe, vocês conseguem criar uma startup digital?”
> 

Os alunos devem estruturar um **produto web real**, mas precisam pensar antes de programar.

Regra importante:

O computador é só uma ferramenta. A estratégia vem primeiro.

## 🔹 Fase 1 – Problema + Ideia

Cada grupo define no papel:

1. Qual problema real da escola vamos resolver?
2. Quem é o usuário?
3. Nome da startup
4. 3 funcionalidades principais

Exemplos que podem surgir:

- Sistema de organização de eventos
- Agenda escolar digital
- Sistema de recados
- Organizador de tarefas escolares

Sempre pensando:

- Isso é simples o suficiente?
- Dá para fazer versão básica em HTML?
- Está claro quem usa?

O que está sendo trabalhado aqui:

✔ Gestão de Startup

✔ Análise de Sistemas

## 🔹 Fase 2 – Protótipo + Estrutura Web

🔁 Rodízio no computador (3 rodadas de 5 minutos)

### No papel:

🎨 UX Designer:

- Desenha 2 telas:
    - Tela principal
    - Tela de cadastro ou listagem

📊 Analista:

- Escreve:
    - 3 requisitos funcionais
    - Fluxo simples (ex: usuário entra → cadastra → visualiza)

👑 Product Owner:

- Garante coerência entre ideia e telas

### 💻 No computador:

Criar:

- Estrutura de pastas (ex: /src, /css)
- Arquivo index.html
- Estrutura básica HTML:
    - header
    - main
    - section
    - form
- Título e descrição do sistema

⚠️ Quem entra no PC continua o que o outro fez.

Regra:

Antes de sair do computador, explicar em 30 segundos o que foi feito.

## 🔹 Organização Final

O grupo revisa:

- As telas desenhadas combinam com o HTML criado?
- As funcionalidades fazem sentido?
- O nome da startup está coerente?

Aqui precisa ser reforçado:

> Sistema bom é sistema organizado.
> 

## 🔹 Pitch Final (2 minutos por grupo)

Cada grupo apresenta:

1. Nome da startup
2. Problema
3. Funcionalidades
4. Como funciona o sistema
5. O que já foi estruturado no HTML
